# BugBomber — AI 编译你的测试工程世界

> **一句话卖点**：输入"系统稳定度95%"，AI 编译出测试规则——你不只是在玩炸弹人，你在模拟测试工程师修复Bug、清理技术债的过程。
>
> **30秒演示流程**：输入一句话 → AI编译3条规则 → 世界色调变化 → 需求变更/研发引入Bug → 放测试探针修复Bug+清理技术债
>
> **AI 挂了也能玩**：无 API Key 时自动启用 6 组应急规则包 + 4 组隐藏组合规则，100% 可演示

## 游戏简介

BugBomber 是一款 **AI 原生** 的测试工程主题炸弹人游戏。玩家输入一句话描述系统状态（如"系统稳定度95%""需求又变更了"），AI 实时编译出 3 条测试规则来驱动整个世界：

- **需求变更系统** — 产品经理定期引入需求变更，增加测试工作量
- **研发引入 Bug 系统** — 研发定期引入各种 Bug，5 种类型各有独特行为
- **技术债累积系统** — 被清理的技术债会按规则重新累积

玩家的任务：用测试探针修复所有 Bug + 清理技术债，使系统稳定度达到 95% = 测试通过，发布成功。

### 五种 Bug 类型

| 类型 | 外观 | 特殊能力 | 修复方式 |
|------|------|----------|----------|
| 🟢 普通 Bug | 绿色虫子 | 正常移动 | 1 个测试探针 |
| 👻 幽灵 Bug | 紫色幽灵 | 可穿墙（回归Bug） | 1 个测试探针 |
| 💙 死机 Bug | 蓝屏方框 | 倒计时崩溃 | 1 个测试探针（要在崩溃前修复） |
| 🔴 P0 Bug | 红色大虫 | 需要修复两次 | 2 个测试探针 |
| 👹 Boss Bug | 线上事故（红犄角+发光红眼+邪笑） | 3血、慢速、每90秒生成 | 3 个测试探针，出场带告警+日志 |

### 道具系统

| 道具 | 效果 |
|------|------|
| 🍗 能量包 | 恢复 1 点生命（测试工程师续命） |
| 🫓 探针+1 | 增加 1 个测试探针上限 |
| 🧋 范围扩大 | 增加测试覆盖范围 |
| ⚡ 加速包 | 10 秒测试执行速度翻倍 |

### 特色亮点

- **AI 规则编译器** — 每局输入不同的话，AI 生成不同的规则，无限重玩价值
- **6 种核心机制规则** — bug_split(Bug分裂) / dark_map(黑盒测试) / one_life(过劳模式) / shishan_regen(屎山自愈) / chain_explosion_boost(技术债爆炸) / test_isolation(测试隔离)
- **开局危险规则全屏警告** — 检测到 one_life/dark_map 等危险规则时，屏幕中央弹出大字警告框，列出致命机制，4秒后自动消失
- **最后1条命红灯警告** — 生命值降至1时，画布边缘红色脉冲呼吸 + 暗角遮罩，营造紧迫感
- **最后一只Bug高亮** — 场上只剩1只Bug时，红色脉冲光环追踪 + 横幅"🎯 最后一个Bug！"
- **一大波 Bug 预警** — 每 45 秒触发"一大波 Bug 正在奔来"横幅，PvZ 风格颤抖放大动画
- **Boss Bug 系统** — 每90秒生成一只3血Boss Bug，出场带红光警告+震动+音效
- **毒舌吐槽系统** — 80+ 条职场毒舌文案，8 种触发场景，打字机效果
- **弹幕系统** — 80+ 条弹幕覆盖击杀/拾取/死亡全场景
- **加速炸屎山特效** — 加速状态下炸屎山，💩 碎片飞出屏幕 + 绿色冲击波
- **开场动画 + TTS 语音** — 像素测试员敲门 → 喊"测试部开工了！Fighting！" → 坐到工位 → 屏幕亮起 → 进入游戏
- **规则面板人话化** — 不再显示工程参数，而是"📢 需求风暴（每6秒改一次需求）"这样的职场黑话
- **三方系统随机抖动** — 产品/研发/屎山触发间隔 ±2s 随机，更像"系统在失控"而非定时器
- **评委首局保护** — 首次游戏自动降低难度（Bug减速20%+道具掉率翻倍），帮评委快速上手
- **CRT 复古滤镜** — FC 像素风 + CRT 扫描线，8-bit 音效

## 玩法

1. **输入一句话** — 如"项目太乱""996""敏捷开发"
2. **AI 编译规则** — AI 将你的描述编译为 3 条游戏规则（3秒超时自动降级到离线规则包）
3. **危险规则警告** — 如果规则含致命机制（过劳/黑盒/测试隔离等），开局全屏大字警告
4. **开始战斗** — WASD/方向键移动，空格放炸弹
5. **目标** — 炸死所有 Bug 怪 + 屎山清除率 ≥ 70%

### 核心数值

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 炸弹爆炸范围 | 3 格 | 技术债爆炸规则下为 4 格 |
| 默认炸弹数 | 2 颗 | 拾取大饼可增加上限 |
| 玩家移动速度 | baseSpeed=4 | 比初版快 60%，Bug速度不变 |
| 初始屎山密度 | 12% | 开局地图清爽 |
| 杀Bug掉屎山概率 | 20% | "改Bug产生新屎山"梗，不会满屏 |
| 屎山总量上限 | 25 块 | 防止地图被填满 |
| 清除率计算 | 初始屎山+产品复活屎山 | 不计杀Bug掉落的屎山，避免死局 |

## 配置 AI 接口（可选）

### 方式一：界面配置（推荐）

1. 打开游戏，在输入界面点击「AI 接口：未配置」展开配置面板
2. 填入 API 地址、API Key、模型名
3. 点击「保存配置」，配置会自动保存到浏览器，下次打开无需重复填写

### 方式二：代码配置

打开 `js/ai/compiler.js`，在文件顶部修改：

```javascript
apiUrl: 'https://api.deepseek.com/v1/chat/completions',
apiKey: 'sk-your-key-here',
model: 'deepseek-chat',
```

**推荐的 API 服务：**

| 服务 | API 地址 | 模型 | 说明 |
|------|----------|------|------|
| DeepSeek | `https://api.deepseek.com/v1/chat/completions` | `deepseek-chat` | 便宜好用，注册送额度 |
| OpenAI | `https://api.openai.com/v1/chat/completions` | `gpt-4o-mini` | 效果好，需海外网络 |
| 智谱 | `https://open.bigmodel.cn/api/paas/v4/chat/completions` | `glm-4-flash` | 国内可用，有免费额度 |

**未配置 API 时**：自动使用离线兜底规则包（6 组预设规则 + 4 组隐藏组合），游戏完全可玩。

## 技术栈

- HTML5 Canvas + 原生 JavaScript（无框架）
